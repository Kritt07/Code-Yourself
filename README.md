## 1) Описание игры (концепция)
**Code Yourself** — пошаговая головоломка с элементами платформера, где игрок **не управляет напрямую**, а пишет набор команд (псевдокод), которые затем выполняются автоматически.

Игрок управляет квадратным персонажем. Цель — пройти уровень, полный препятствий (пилы, движущиеся объекты, ловушки), используя заранее прописанный алгоритм действий.

Ключевая идея: игрок должен **не реагировать**, а **предугадывать** и **программировать** решение.

Каждый уровень — это задача на:
- логику
- тайминг
- оптимизацию команд

## 2) Основные механики

### 2.1) Пошаговая система (Tick-based)
- Игра работает по тикам (шагам времени).
- 1 тик = фиксированное время (например, 1 секунда).
- На каждом тике:
  - выполняется команда
  - обновляется мир (препятствия)
  - проверяются столкновения

### 2.2) Программирование действий
- Игрок пишет команды в текстовом поле и нажимает **Run**.
- Во время выполнения на каждом тике подсвечивается текущая выполняемая команда. Это помогает:
  - понимать поведение персонажа
  - находить ошибки
  - обучаться логике игры

### 2.3) Система команд
Базовые команды:
- `MOVE LEFT n`
- `MOVE RIGHT n`
  - движение на `n` шагов (по тикам)
- `JUMP LEFT`
- `JUMP RIGHT`
  - один прыжок = 1 тик, траектория — парабола с высотой **120px** (**без физики**)
- `WAIT n`
  - ожидание `n` тиков

Циклы:
- `REPEAT n`
  - блок команд
- `END`
  - закрытие блока

Повтор блока команд `n` раз предполагается реализовать через **стек**.

### 2.4) Ограничения
- лимит на количество строк (например, 5–10)
- ограниченное количество команд

### 2.5) Препятствия (паттерны)
Все препятствия работают по детерминированным паттернам и синхронизированы с тик‑системой.

Примеры:
- пила (движется туда‑обратно)
- молот (опускается каждые `N` тиков)

### 2.6) Плавность (визуал)
Логика дискретная (по тикам), но визуал должен быть плавным. Реализуется через интерполяцию:
- позиции между тиками анимируются
- создаётся ощущение “живого” движения

## 3) Управление и интерфейс
Игровое:
- **Run** — запустить выполнение
- **Reset** — перезапустить уровень

Интерфейс/макет:
- левый блок — редактор команд
- правый блок — игровое поле

## 4) Архитектура (высокоуровнево)
Сущности/модули (названия могут отличаться в проекте, но смысл такой):
- **Command** — базовый класс/тип команд
- **Parser** — разбор текста в структуру команд (включая вложенные блоки `REPEAT`)
- **Executor** — выполнение команд по тикам (курсор текущей команды, обработка `n`, стек повторов)
- **GameState** — текущее состояние мира (позиции, объекты)
- **Renderer** — отрисовка/анимация (интерполяция между тиками)

## 5) План разработки

### Неделя 1
Цель: сделать “игра вообще работает”.

Обязательно:
- окно приложения
- базовая отрисовка (персонаж)
- тик‑система
- простое движение (без команд)

Результат: персонаж двигается по тикам.

### Неделя 2
Цель: реализовать программирование.

Задачи:
- парсер команд
- выполнение команд
- UI для ввода
- тесты

Результат: игрок может пройти уровень через код.

### Неделя 3 (возможны изменения)
Цель: геймплей.

Задачи:
- препятствия
- паттерны движения
- столкновения
- баланс тиков
- тесты

### Неделя 4 (возможны изменения)
Цель: контент и полировка.

Задачи:
- сделать победу и поражение
- анимации

### Неделя 5
Задачи:
- уровни
- меню
- тестовый интерфейс игры

## 6) Definition of done (уровень/прогон)
- Команды воспроизводимо выполняются по тикам (детерминизм).
- Подсветка текущей команды соответствует фактическому исполнению.
- Мир обновляется и проверяет столкновения на каждом тике.
- `Reset` полностью возвращает состояние уровня к стартовому (включая паттерны препятствий и позицию персонажа).
